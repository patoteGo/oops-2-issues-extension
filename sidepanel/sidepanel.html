Settings

GitHub token used to list repositories, upload screenshots, and create issues.

Which scopes does the token need?

Create one at github.com/settings/tokens.

Fine-grained — pick the repos you'll file against, then enable:

  • Issues: Read and write — create issues
  • Contents: Read and write — upload screenshots (the public-assets fallback) + auto-create <you>/oops-assets
  • Metadata: Read — required, auto-included

Classic — a single repo scope covers all of the above (public + private).

Screenshots also try GitHub's session attachment first (no token needed for that part) — so even a read-mostly token still gets inline images if you're logged into github.com.

Screenshots try to attach via your github.com session first (renders inline on public and private repos). If that is unavailable, they fall back to a public repo whose raw URL still renders inline. Leave blank to auto-create <you>/oops-assets.
